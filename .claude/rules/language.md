# 작성 언어 규칙
이 프로젝트의 문서와 로그는 **한글로 작성**한다.

**한글로 작성하는 대상**
- 마크다운 문서(`*.md`): CLAUDE.md, `.claude/rules/`, README 등.
- 에이전트용 문서: `.claude/` 아래의 규칙·가이드.
- 코드의 로그 메시지: `console.log` / `console.error` 등 개발자용 출력 문자열.
- 주석 중 설명이 필요한 본문(짧은 인라인 주석은 상황에 맞게).
- 커밋 제목(subject)의 설명부 — 커밋 규칙(@commits.md)을 따른다.

**영어를 유지하는 대상**
- 코드 식별자: 변수·함수·타입·파일·경로명.
- 커밋의 type·scope 토큰(`feat`, `fix`, `A-3` 등) — 커밋 규칙(@commits.md)을 따른다.
- 외부 계약 값: 스키마 키, enum 값, env 키, 커맨드명(`/start` 등).
- 봇이 텔레그램으로 보내는 사용자 응답 문자열(제품 UX) — 변경은 별도 결정.

**원칙**
- 새 문서나 로그를 추가할 때 기본 언어는 한글이다.
- 기술 용어는 굳이 번역해 어색해지면 영어 그대로 둔다(예: producer, contract,
  schema, slug). 자연스러움을 우선한다.
