# TODO 관리

프로젝트의 진행 상황과 남은 할일은 루트 `TODO.md` 한 곳에서 관리한다.
이 스킬은 레포 상태(로드맵, 코드, 커밋)와 `TODO.md`를 대조해 갱신한다.

## 절차
1. `TODO.md`를 읽는다(없으면 새로 만든다).
2. `CLAUDE.md`의 `## Roadmap`과 실제 코드 상태를 비교해 완료/미완료를 판정한다.
3. 변경 사항을 반영한다:
   - 끝난 항목은 `- [x]`로 체크(지우지 않고 이력으로 남긴다).
   - 새로 발견한 이슈·기술 부채·운영 전 처리 항목을 추가한다.
   - 결정이 필요한 항목은 `## 미정 (결정 대기)`에 근거·선택지와 함께 적는다.
4. 무엇을 바꿨는지 사용자에게 요약 보고한다.

## 형식
- 한글로 작성한다(@.claude/rules/language.md).
- 체크박스 목록: 미완료 `- [ ]`, 완료 `- [x]`.
- 섹션으로 묶는다: `## 로드맵`, `## 알려진 이슈`, `## 미정 (결정 대기)`.

## 원칙
- 완성도·남은 할일에 대한 메모는 대화에만 남기지 말고 항상 `TODO.md`에 반영한다.
- `TODO.md`를 단일 출처로 유지한다 — 다른 곳에 흩어진 할일 목록을 만들지 않는다.
